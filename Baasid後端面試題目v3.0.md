# Baasid 後端工程師面試題目 v3.0

## 情境

`starter-project/` 是一個**商品交易平台**的後端，由前一位開發者交接給你。平台有兩種角色：**管理員（ADMIN）** 可上架／修改／下架商品；**一般用戶（USER）** 可瀏覽商品並下單。

前一位開發者交接時表示專案「能啟動、主要流程能跑」，但他趕工、又大量依賴 AI 生成，留下不少坑。**你的任務是接手它、讓它變成一個可上 production 的服務。**

### 技術棧（沿用，請勿整包重寫換語言）
- Java 21 / Spring Boot 4.0
- Spring Security + JWT
- Spring Data JPA / PostgreSQL
- Swagger (springdoc)

### 如何啟動
```
cd starter-project
docker compose up -d        # 啟動 PostgreSQL
./mvnw spring-boot:run      # 啟動應用程式
```
Swagger UI： http://localhost:8080/swagger-ui.html
預設帳號：`admin / admin123`（ADMIN）、`alice / alice123`（USER）

## 你的任務

### 第一部分：Code Review 報告（必繳）

通讀整個專案，產出一份 **Code Review 報告**（Markdown）。請：

1. **列出你發現的問題**，每一項註明：
   - 問題描述（在哪個檔案／哪一段）
   - 為什麼這是問題（會造成什麼後果）
   - 嚴重程度：**🔴 嚴重 / 🟡 中等 / 🟢 輕微**
2. **依嚴重程度排序**，把最該優先處理的放前面。
3. 如果你判斷某個寫法「看起來怪但其實是合理取捨」，也請說明你為什麼決定**不**改它。

### 第二部分：實際修正（必繳）

根據你的報告，動手修正。要求：

- **所有你標為 🔴 嚴重的問題都必須修好。**
- 🟡 中等問題請量力而為，挑你認為最有價值的修。
- **用 git 管理你的修正**，commit 粒度與訊息要能讓我們看出你的思路（請自行 `git init` 並從一個 baseline commit 開始）。
- 修正要有**測試或其他方式佐證**「真的修好了」，特別是併發相關的修正。

### 第三部分：擴充一個小功能（必繳）

在你整理過的程式碼基礎上，**從你下面挑一項實作**（也可自行提議，先說明理由）：

- **A.** 商品查詢進階功能：模糊搜尋 + 價格區間 + 排序 + 分頁，並考慮百萬級資料量的效能。
- **B.** 操作日誌（Audit Log）：記錄商品的新增／修改／刪除，含操作者、時間、變更前後差異，並提供查詢 API。

### 第四部分：AI 使用說明（必繳）

誠實寫一段說明：

- 這份作業你哪些部分借助了 AI？（讀程式碼、找 bug、寫修正、寫測試……）
- **你如何驗證 AI 的產出？** 例如：AI 有沒有「自信地改錯」、有沒有引入新的 bug、你怎麼發現的？
- 有沒有哪次你**否決**了 AI 的建議？為什麼？
